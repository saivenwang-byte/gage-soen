# 05 · 数据字段说明（一条优惠 deal）

## 核心字段（前后端共用，见 normalizeDeal.js）

| 字段 | 含义 |
|------|------|
| id | 唯一标识 |
| scene | 场景：stay / coffee / pet / expiring / entertainment / life / shopping / dining |
| subCategory | 子类 key，对齐 sceneTaxonomy |
| merchantName / title | 店名 |
| address | 地址 |
| lat / lng | 坐标（无则无法地图/串联） |
| promoText / discountText | 优惠文案 |
| price / discountPrice / originalPrice | 价格 |
| imageUrl | 实景图 URL |
| tags / emotionTags | 环境/特色标签 |
| bloggers[] | { nickname, summary, likes } |
| platformLabel | 如「小红书精选」「家人上传」 |
| dataMode | curated / ugc / fallback / real |
| officialUrl / sourceUrl | 可选，详情外链比价 |
| distanceKm / distanceText | 与用户或地图中心的直线距离 |

## 店家介绍（前端生成）

若无 `merchantIntro`，由 `buildMerchantIntro()` 从 promo、town、博主语拼接。  
`services[]` 由 `buildMerchantServices()` 从优惠、价格、标签、地址生成列表。

## 种子数据位置

- 前端：`源码-前端/data/dealsSeed.json`  
- 后端：`源码-后端/data/deals.seed.json`  
（两处应同步）

## UGC 提交字段

- scene, merchantName, address, promoText, price, submitter（昵称，非商家承包）
