# 奉贤区团购比价小程序 — 可行性说明

## 结论：**可以做**，建议分阶段落地

| 能力 | 可行性 | 说明 |
|------|--------|------|
| 地域限定（奉贤区） | ✅ | 经纬度/行政区字段过滤即可 |
| 人数/日期/分类筛选 | ✅ | 业务规则清晰，后端可算「人均实付」 |
| 团购规则解析（N人免M人、满减） | ✅ | 正则 + 结构化字段，见 `backend/app/promo_parser.py` |
| 多品类比价（吃/住/行/玩） | ✅ | 统一 `Deal` 模型 + 分类标签 |
| 打开即搜 / 定时刷新 | ✅ | 小程序 onShow 拉 API；后端定时任务增量同步 |
| 全自动深度爬取主流平台 | ⚠️ | 美团/点评/携程有反爬与 ToS 限制，需适配器 + 合规策略 |

## 推荐数据策略（三轨并行）

1. **MVP（当前）**：内置奉贤示例数据 + 可插拔爬虫适配器，验证筛选与比价逻辑。
2. **半自动**：运营在后台录入/校正链接，定时用 Playwright 只读抓取公开详情页。
3. **长期**：对接平台开放能力（美团联盟、携程分销等）或官方团购 API。

## 法律与合规提醒

- 仅抓取**公开页面**、控制频率、注明来源链接。
- 小程序上架需企业主体、类目资质（旅游/餐饮等视类目而定）。
- 不在产品内存储用户支付信息；跳转平台 H5/小程序完成下单。

## 技术架构

```
miniprogram/     微信小程序（筛选、列表、比价详情）
backend/         FastAPI + SQLite（搜索、比价排序、同步任务）
  adapters/      美团/点评/携程等爬虫适配器（可逐个启用）
```

## 比价核心指标

- **人均实付** = 根据人数优惠规则折算后的每人应付
- **节省金额** = 原价人均 − 实付人均
- **优惠力度分** = 用于列表排序的综合分（节省比例 + 是否团购 + 评分）
